Relative Reference :
	Cell reference that changes automatically when you copy or drag the formula to another cell.
	Example:
		In cell C1 → =A1 + B1
		If you copy it to C2, it becomes =A2 + B2
	Use Case: Helpful when you want the formula to adjust row/column automatically (like summing each row).

Absolute Reference :
	Cell reference that does not change when you copy or drag the formula. It is "locked" using $.
	Example:
		In cell C1 → =A1 * $B$1
		If you copy it to C2, it remains =A2 * $B$1 (column B row 1 stays fixed).
	Use Case: Helpful when you always want to use a fixed value, like a tax rate or discount stored in one cell.

Mixed References :
	A cell reference where either the row or the column is fixed (with $), but not both.
	It’s a “mix” of relative + absolute.

	Types of Mixed References
		$A1 → Column A is fixed, row changes when copied.
		A$1 → Row 1 is fixed, column changes when copied.

	Example :
		if we have 5 columns and 100 rows..1st column and row has values....using this B2=$A2*B$1.....to fill other empty columna nd rows...
		|   | A | B | C | D |
		| 1 | 1 | 2 | 3 | 4 | 
		| 2 | 2 |   |   |   |
		| 3 | 3 |   |   |   |
		| 4 | 4 |   |   |   |


IF --> Tests one condition and returns one value if TRUE, another if FALSE.
	Example ....=IF(A1>50, "Pass", "Fail")...If A1 > 50 → shows Pass, else Fail.

Nested IF --> Nested IF is when we use multiple IFs inside each other to handle more than one condition.Useful if you don’t have IFS (older Excel versions).
	Example...=IF(A1>=90,"A", IF(A1>=75,"B", IF(A1>=50,"C","Fail")))

IFS --> Tests multiple conditions in one formula. Returns the value for the first TRUE condition.
	Example.....=IFS(A1>=90,"A", A1>=75,"B", A1>=50,"C", A1<50,"Fail")->If A1 = 85 → shows B.
				(OR)
	Example.....=IFS(A1>=90,"A", A1>=75,"B", A1>=50,"C",True,"Fail")

Maxifs --> Returns the largest value that meets conditions.
	Example ....=MAXIFS(C2:C10(Sales column), A2:A10(Region Column), "East")...Finds the maximum Sales in the East region.

Minifs --> Returns the smallest value that meets conditions
	Example ..=MINIFS(C2:C10(Sales column), A2:A10(Region Column), "East")...Finds the minimum Sales in the East region.

Sumifs --> Adds numbers that meet multiple conditions.
	Example...=SUMIFS(C2:C10(Sales column), A2:A10(Region Column), "East", B2:B10(product column), "Apple")..Adds Sales (C2:C10) where Region is East and Product is Apple.

Countifs --> Counts how many rows meet multiple conditions.
	Example ...=COUNTIFS(A2:A10(Region column),"East", B2:B10(Product column),"Apple")......Counts how many times Apple was sold in East region...

Iferror --> Returns a custom value if a formula gives any error (like #N/A, #DIV/0!, #VALUE!, etc.).
	Example ....=IFERROR(A1/B1, "Error")....If B1=0, Excel normally shows #DIV/0!. With IFERROR, it shows "Error" instead....

Ifna --> Returns a custom value only if the error is #N/A (not other errors).
	Example ...=IFNA(VLOOKUP(101, A2:B10, 2, FALSE), "Not Found")...If ID 101 is not in the table, Excel shows "Not Found" instead of #N/A.

VLOOKUP--> looks down the first column of a table for a value and returns a value from a specified column in the same row.
	=VLOOKUP(lookup_value, table_array, col_index_num, [range_lookup])

HLOOKUP --> looks across the first row of a table for a value and returns a value from a specified row in the same column.
	=HLOOKUP(lookup_value, table_array, row_index_num, [range_lookup])

XLOOKUP --> modern, flexible lookup that searches a lookup_array and returns the corresponding value from a return_array. It can search left/right/up/down, handle not-found results, and return arrays.
	=XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found], [match_mode], [search_mode])

Data Validation --> Data Validation in Excel is a feature that controls what kind of data can be entered into a cell.
		It helps prevent wrong or invalid data entry.

Index and Match --> 
	INDEX + MATCH is used for powerful lookups. MATCH finds the row/column position of a value, and INDEX returns the value at that position. 
	It’s more flexible and reliable than VLOOKUP.
	=INDEX(C2:C4, MATCH(102, A2:A4, 0))

two-way lookup --> “Using two MATCH functions — one for the row and one for the column — with INDEX allows us to do a two-way lookup. 
	This is more powerful than VLOOKUP or HLOOKUP, which only search in one direction.”
|       | B (Jan) | C (Feb) | D (Mar) |
| ----- | ------- | ------- | ------- |
| **A** |         |         |         |
| 101   | 200     | 250     | 300     |
| 102   | 150     | 180     | 210     |
| 103   | 400     | 420     | 500     |

	=INDEX(B2:D4, MATCH(102, A2:A4, 0), MATCH("Feb", B1:D1, 0))
